  1. IDLink Glossary
    1. Registered user: All users registered on the company side in IDLink
    2. System Owner: The person who has created the company in Safe Profile. The owner is also always an administrator. Ownership (incl. Administrator) can be transferred to another named person. There will therefore always be an Owner (and an Administrator). This secure that we will never be in a “dead-lock”
    3. System Administrator: The person(s) that is the system administrator (setup, user management) There can be more than one Administrator
    4. System Manager: The person who setup system-functionality – and templates/input forms. Apart from this a System-manager are able to perform the same tasks as a System-user
    5. System-user: The person who process incoming requests and creates responses
    6. System-manager: The person who setup system-functionality – and templates/input forms. Apart from this a System-manager are able to perform the same tasks as a System-user
    7. Consumer: Requesting and receiving data.
    8. Customer Data: Is defined as all data that is collected in an incoming request in IDLink, and sent to the requester
    9. Data Requester: Is an individual that has sent a data request OR the individual that has given consent that data can be requested on his/her behalf
    10. Data Recipient: Is the individual or organisation that will receive the data.
    11. Request owner: A system user can be a request owner. This happens when he/she is assigned to an incoming request. The ownership of a request can be passed on to another system user if needed
    12. Request Approver: A Request Approver is a Registered user that is assigned by the administrator or by the Request owner, to ensure that the customer data that has been added to the request is correct and sufficient before sending to the data recipient
